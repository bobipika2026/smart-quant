# 因子矩阵系统文档

## 概述

因子矩阵系统是一个基于01矩阵设计的多因子策略回测与评价系统，通过组合不同策略因子、条件因子和时间因子，寻找最优的策略组合。

---

|----------|----------|------|------|

| ma_10_30 | MA金叉(10,30) | short=10, long=30 | 趋势 |
| macd_default | MACD(12,26,9) | fast=12, slow=26, signal=9 | 趋势 |
| macd_fast | MACD快速(8,17,9) | fast=8, slow=17, signal=9 | 趋势 |
| rsi_14_70 | RSI(14,超买70) | period=14, upper=70, lower=30 | 震荡 |
| rsi_14_80 | RSI(14,超买80) | period=14, upper=80, lower=20 | 震荡 |

## 二、互斥规则

### 2.1 策略因子互斥组

同一策略的不同参数版本互斥，只能选其中一个：

| 互斥组 | 包含因子 | 选择方式 |
|--------|----------|----------|
| MA组 | ma_5_20, ma_5_30, ma_10_20, ma_10_30 | 4选0或1 |
| MACD组 | macd_default, macd_fast | 2选0或1 |
| RSI组 | rsi_14_70, rsi_14_80 | 2选0或1 |

### 2.2 条件因子互斥组

同一字段的不同阈值互斥：

| 互斥组 | 包含因子 | 选择方式 |
|--------|----------|----------|
| PE组 | pe_lt_20, pe_lt_15 | 2选0或1 |
| ROE组 | roe_gt_10, roe_gt_15 | 2选0或1 |
| North组 | north_gt_1, north_gt_3 | 2选0或1 |
| Cap组 | cap_gt_100, cap_gt_500 | 2选0或1 |

### 2.3 非互斥因子

以下因子可以自由组合：
- KDJ、布林带、CCI、WR（策略因子）
- PB、负债率（条件因子）

## 三、组合计算

### 3.1 组合数量计算

```
策略组合数 = (MA选法) × (MACD选法) × (RSI选法) × (非互斥策略选法)
          = 5 × 3 × 3 × 2^4
          = 720种

条件组合数 = (PE选法) × (ROE选法) × (North选法) × (Cap选法) × (非互斥条件选法)
          = 3 × 3 × 3 × 3 × 2^2
          = 324种

时间组合数 = 4种（必选1个）

总组合数 = 720 × 324 × 4 = 931,200种
```

### 3.2 有效组合（排除无意义组合）

**排除规则：**
1. 至少选1个策略因子
2. 必须选1个时间因子
3. 条件因子可选可不选

**有效组合数：**
```
有效策略组合 = 720 - 1 = 719种
有效总组合 = 719 × 324 × 4 = 929,304种
约 93万种
```

## 四、回测逻辑

### 4.1 信号组合方式

当多个策略因子被选中时，使用OR模式组合信号：

```python
# 信号取值：1=买入, 0=持有, -1=卖出
combined_signal = clip(sum(all_signals), -1, 1)
```

**OR模式含义：**
- 任一策略发出买入信号 → 执行买入
- 任一策略发出卖出信号 → 执行卖出

### 4.2 回测流程

```
1. 获取历史数据（根据时间因子确定数据范围）
2. 计算每个策略因子的信号序列
3. 组合信号（OR模式）
4. 执行回测模拟
5. 计算各项指标
```

### 4.3 交易规则

- **买入条件**：信号=1 且 有现金
- **卖出条件**：信号=-1 且 有持仓
- **手续费**：0.03%（双边）
- **仓位管理**：全仓买入/卖出

## 五、评价体系

### 5.1 收益类指标

| 指标 | 公式 | 说明 |
|------|------|------|
| 总收益率 | (期末-期初)/期初 × 100% | 整体收益 |
| **年化收益率** | (期末/期初)^(1/年数) - 1 | 标准化收益，**用于排序** |
| 基准收益 | (期末价-期初价)/期初价 | 买入持有收益 |
| 超额收益 | 策略收益 - 基准收益 | 跑赢基准 |

### 5.2 风险类指标

| 指标 | 公式 | 说明 | 优秀标准 |
|------|------|------|----------|
| 最大回撤 | max(峰值-谷值)/峰值 | 最大亏损 | <15% |
| 波动率 | 策略净值日收益标准差 × √252 | 策略波动程度 | 越低越好 |
| 下行风险 | 负收益标准差 × √252 | 亏损风险 | 越低越好 |

### 5.3 风险调整指标

| 指标 | 公式 | 说明 | 优秀标准 |
|------|------|------|----------|
| 夏普比率 | (收益-无风险利率)/波动率 | 单位风险收益 | >1.0 |
| 索提诺比率 | (收益-无风险利率)/下行风险 | 只惩罚下行 | >1.0 |
| 卡玛比率 | 年化收益/最大回撤 | 收益回撤比 | >1.0 |

### 5.4 交易质量指标

| 指标 | 说明 | 优秀标准 |
|------|------|----------|
| 胜率 | 盈利交易/总交易 | >50% |
| **盈亏比** | 平均盈利/平均亏损 | **>1.5** |
| 交易次数 | 期间交易总数 | 适中 |

### 5.5 综合评分（满分100分）

| 评分项 | 权重 | 评分标准 |
|--------|------|----------|
| **年化收益率** | 40分 | >30%=40分, >20%=35分, >10%=25分, >0%=15分 |
| 夏普比率 | 30分 | >2=30分, >1=25分, >0.5=15分, >0=10分 |
| 最大回撤 | 20分 | <10%=20分, <15%=18分, <20%=15分, <30%=10分 |
| 胜率 | 10分 | >60%=10分, >50%=8分, >40%=5分 |

### 5.6 评级标准

| 评级 | 分数范围 | 说明 |
|------|----------|------|
| A+ | ≥80分 | 极优 |
| A | ≥70分 | 优秀 |
| B+ | ≥60分 | 良好 |
| B | ≥50分 | 一般 |
| C | ≥40分 | 较差 |
| D | <40分 | 不推荐 |

## 十二、回测引擎统一（重要）

### 12.1 问题背景

在开发过程中，出现了两套回测引擎：
1. **BacktestEngine**（回测中心使用）
2. **VectorizedFactorBacktest**（向量化引擎）

初期两套引擎的信号生成逻辑和回测计算方式不同，导致同一股票、同一策略的回测结果不一致。

### 12.2 问题表现

| 引擎 | MACD快速(000001) | 差异原因 |
|------|------------------|----------|
| 向量化引擎 | 32.67% | 信号：事件驱动（只在交叉发出） |
| 回测引擎 | 20.74% | 信号：状态驱动（持续发出状态） |

### 12.3 解决方案

**核心原则：统一信号生成和回测计算**

```python
# 向量化引擎使用回测引擎的策略类
def generate_macd_signals(self, fast=12, slow=26, signal=9):
    from app.services.strategy import MACDStrategy
    df = self.df.copy()
    strategy = MACDStrategy(fast=fast, slow=slow, signal=signal)
    df_signal = strategy.generate_signals(df)
    return df_signal['signal'].values

# 向量化引擎使用BacktestEngine计算收益
def backtest_strategy(self, signals):
    from app.services.backtest import BacktestEngine
    df = self.df.copy()
    df['signal'] = signals
    engine = BacktestEngine()
    return engine.run_backtest(df)
```

### 12.6 经验教训

1. **单一数据源原则**
   - 所有回测必须使用同一个BacktestEngine
   - 所有策略信号必须使用同一个策略类库
   - 避免重复实现导致不一致

2. **信号类型统一**
   - 状态驱动：信号=1表示持仓，信号=-1表示空仓
   - 不要混合使用事件驱动（只在交叉发出信号）

3. **信号生成顺序（重要）**
   - **错误**：先对全部数据生成信号，再截取前N天回测
   - **正确**：先截取数据，再对截取后的数据生成信号，然后回测
   - 原因：技术指标（MA、MACD等）依赖历史数据，截取后重新计算结果会不同

4. **时间范围对齐**
   - 向量化引擎按时间因子(period_3m/6m/1y/2y)分别回测
   - 回测中心默认使用全部数据
   - 对比时必须使用相同时间范围

5. **测试验证**
   - 任何新引擎必须与标准引擎对比验证
   - 使用相同股票、相同策略、相同时间范围，结果必须完全一致

6. **代码复用**
   - 向量化引擎不重新实现策略逻辑
   - 直接调用 `app.services.strategy` 中的策略类
   - 直接调用 `BacktestEngine` 计算收益

## 十四、专业因子评分系统

### 14.1 系统概述

专业因子评分系统基于 Barra/MSCI 等专业量化机构因子框架，采用五大风格因子模型：

```
┌─────────────────────────────────────────────────────────┐
│                综合评分 (Composite Score)                │
│                      100分制                            │
└─────────────────────────────────────────────────────────┘
                          │
    ┌─────────┬─────────┬──┴──┬─────────┬─────────┐
    ▼         ▼         ▼     ▼         ▼         ▼
┌───────┐ ┌───────┐ ┌───────┐ ┌───────┐ ┌───────┐
│ 价值  │ │ 成长  │ │ 质量  │ │ 动量  │ │ 情绪  │
│ 25%   │ │ 20%   │ │ 25%   │ │ 15%   │ │ 15%   │
└───────┘ └───────┘ └───────┘ └───────┘ └───────┘
```

### 14.2 五大风格因子

| 风格 | 权重 | 细分因子 | 核心逻辑 |
|------|------|----------|----------|
| **价值** | 25% | EP、BP、SP、NCFP、股息率 | 寻找被低估的便宜好公司 |
| **成长** | 20% | 营收增长、利润增长、ROE变化、增长加速度、EPS增长 | 挖掘高增长潜力股 |
| **质量** | 25% | ROE、ROA、毛利率、净利率、财务杠杆、流动比率、应计项目 | 筛选财务健康的优质公司 |
| **动量** | 15% | 价格动量、相对强度、成交量动量、均线偏离、突破形态 | 跟随趋势强者恒强 |
| **情绪** | 15% | 北向资金、换手率、量比、融资余额、波动率 | 捕捉市场情绪和资金流向 |
